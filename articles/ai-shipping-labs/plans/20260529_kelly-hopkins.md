# Plan: Kelly Hopkins

Internal working document. Share only the `Summary` and `Plan` sections with the member.

## Summary

- Current situation: about 8 years in traditional data analytics and data science, a beginner with AI and AI tools, just finished a grad program at NYU and a fellowship, now ready to shift toward AI this summer.
- Goal for the next 6 to 8 weeks: build a solid foundation in AI and learn how it fits into data work, while doing small builds along the way.
- Main gap to close: an unclear learning path and too many online resources, which lead to analysis paralysis before she can start.
- Weekly time commitment: 5 to 10 hours per week, including at least one hour per workday for learning and development.
- Why this plan is the right next step: LLM Zoomcamp starts in about a week and a half and gives her exactly the structured path she is missing - one clear course to follow, with RAG, tools, and agents covered in order, plus a project to build.

## Plan

This plan is built around two sprints, as Kelly asked. The first sprint is the foundational one: take LLM Zoomcamp, learn the fundamentals, and build a project inside the course. The second sprint ramps up to a more advanced build, with the direction chosen at the end of the first sprint based on what she wants to go deeper into.

## Focus

- Main focus: take part in LLM Zoomcamp and build a project inside the course. The course is the structured path that cuts through the analysis paralysis.
- Supporting focus: start using an AI coding assistant now as an accelerator, not as a replacement for writing code. Read and try to understand the code it produces.
- Supporting focus: keep each week shippable - one small artifact or demo per week, since that is what keeps Kelly on track.

## Timeline

The course starts in about a week and a half. The weeks below cover a short prep period before it starts, then a six-week first sprint aligned to the course. The second sprint is sketched out but planned in detail at the end of the first one.

Pre-sprint prep (now until the course starts, about a week and a half):

- Open the first lesson of LLM Zoomcamp. The text version is already available, so there is no need to wait for the video. Skim it and gauge how hard the code feels - just look for constructs that are unclear, no need to actually do the exercises yet.
- If anything in that first lesson feels rusty on the Python side, use this window to brush up specifically on what came up. This is the time to pull up any gaps before the course starts.
- If the first lesson reads clearly, she can start working through the material now, since it is already published, rather than waiting for the official start.
- Pick an AI coding assistant and start using it. Since she already uses ChatGPT and has a paid subscription, Codex is already included, so that is a natural place to start. Use it as a productivity multiplier, and pay attention to the code it writes.

Sprint 1 - LLM Zoomcamp (six weeks, main focus):

Week 1:
- Start the course with the rest of the cohort. Work through the first module and its homework.
- Post one small artifact or demo at the end of the week.

Week 2:
- Continue with the next module and homework, building on week 1.
- One artifact or demo at the end of the week.

Week 3:
- Continue with the course modules and homework. RAG, tools, and agents are all covered here, so this is where the AI fundamentals come together.
- One artifact or demo at the end of the week.

Week 4:
- Continue with the course modules and homework.
- Start scoping the course project - a small, beginner-friendly build that uses what the course has covered so far.
- One artifact or demo at the end of the week.

Week 5:
- Build the course project. This is the project she will need to complete for the course.
- One artifact or demo at the end of the week.

Week 6:
- Finish and present the course project.
- Wrap up the sprint and decide the direction for sprint 2.

Sprint 2 (planned in detail at the end of sprint 1):

The next AI Shipping Labs sprint runs in parallel with the course, so the members taking the course do their sprint alongside it. After the course, the direction depends on what Kelly wants to go deeper into. There is no need to decide this far ahead.

- Option A: another project in the same style as the Zoomcamp project, extending what she already built or starting a fresh one.
- Option B: something adjacent - for example more focus on evaluation, more focus on monitoring, or a related but different area.
- Either way, the second-sprint project can become a work prototype and a foundation for a larger AI project later, which is what she wants from the final project.
- If she can set aside a full day for the second sprint, she has two good shapes: a small piece of practice each day, or a single focused topic with a clear, self-contained project in it.
