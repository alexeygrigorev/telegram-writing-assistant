# User Interviews

Overview of the user interview effort for AI Shipping Labs. Individual interviews are stored one-file-per-participant at [interviews/](interviews/_index.md). Each interview file includes a persona assignment based on [personas.md](personas.md).

## What We Learned Across Interviews

The strongest recurring theme across the first round of calls and survey responses: members already have plenty of content. The problem is accountability, not content. This applies especially to participants of the AI Engineering Buildcamp (first or second cohort). See the [anonymous Buildcamp participant interview](interviews/anonymous-buildcamp-participant.md) and the [Jakob Zischka interview](interviews/jakob-zischka.md) for the two most detailed accounts.

When people who haven't bought the course start joining the community, they will probably have slightly different needs. But for now there aren't many of those, so we talk to each one individually and create a plan for them[^5].

## How Interviews Are Organised

Individual interview notes now live at [interviews/](interviews/_index.md) - one file per participant. When new interview content comes in (a call transcript, written answers, or a survey response), add it to the existing file for that person or create a new file in that folder[^36].

Each interview file assigns a persona from [personas.md](personas.md). If there is not enough information yet, the persona is marked Undetermined[^36].
