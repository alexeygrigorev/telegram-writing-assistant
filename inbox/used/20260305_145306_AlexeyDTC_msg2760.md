Практика “50 проблем за 30 минут”

1. Выбрать 2 домена (например: хобби + работа).


2. По каждой линзе набросать минимум 3 пункта.


3. Не обсуждать решения. Только проблемы.


4. Потом кластеризовать: повторяющиеся объединить.

3) Trigger Library: когда человек застрял (и как быстро его раскачать)

Trigger Library — это не “идеи проектов”, а триггеры, которые дают мозгу зацепку. Я дам и триггеры, и примеры “типов проектов”, чтобы можно было сразу подсказать направление.

Триггер 1 — “Проведи инвентаризацию данных”

Вопросы

Какие данные у тебя уже есть? (файлы, заметки, трекеры, логи, истории)

Какие данные ты регулярно создаешь?

Какие данные легко добыть из интернета по твоему хобби?


Типы проектов

дашборд/аналитика

прогноз/скоринг

anomaly detection

поиск/индексация



---

Триггер 2 — “Автоматизация рутины”

Вопросы

Что ты делаешь руками каждую неделю?

Что ты копируешь между инструментами?

Какие “мини-процессы” повторяются?


Типы проектов

ETL пайплайн

интеграции через API

workflow automation

“бот-ассистент” для рутины



---

Триггер 3 — “Поиск и навигация по знаниям”

Вопросы

Где ты чаще всего что-то ищешь?

Какие документы/чаты/материалы трудно “прокопать”?


Типы проектов

semantic search

RAG / Q&A по личным данным

суммаризация + выделение action items

knowledge graph (в легкой версии)

Типы проектов

Мини-библиотека “универсальных проектных форм”

Это прям готовые “скелеты”, которые агент может предлагать:

1. Data pipeline → dashboard


2. Search / semantic search по корпусу


3. RAG assistant по личным/доменных данным


4. Recommender / ranking


5. Classifier (тема/тон/жанр/тип)


6. Extractor (из PDF/веба/почты) → структурированные данные


7. Anomaly detection / мониторинг


8. Forecasting / prediction


9. Clustering / segmentation


10. Optimization / scheduling




---

Как это упаковать в агента (очень практично)

Состояние (state), которое агент хранит

domains[]

problems_raw[]

problems_shaped[]

ideas[]

finalists[]

constraints (time, data, tooling)

homework_log


Правила поведения

1–2 вопроса за ход

каждые 5–7 сообщений: краткое резюме + что дальше

если “не знаю” → включить Homework Loop или Trigger Library
