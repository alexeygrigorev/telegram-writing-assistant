Ну смотри, end-to-end это когда есть front-end, back-end и база данных, да, и это все куда-то
деплоится. Вот это end-to-end. У нас, в принципе, это есть и деплой на GitHub Pages. У нас просто в
качестве, у нас нет back-end, у нас все статически генерируется через Jekyll. То есть у нас есть
файлы в Git, и из них этот static generator, ну этот Jekyll генерирует HTML. HTML уже деплоится на
GitHub Pages, то есть у нас, по сути, бэкэнда как такового нету, есть только уже сгенерированный
статический HTML-страницы. Это очень классно для сайтов типа блога, потому что не нужно динамики
никакой, то есть не нужно из баз данных что-то подгружать, и поэтому это можно бесплатно делать.
