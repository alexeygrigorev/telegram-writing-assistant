What AI Engineering Buildcamp is
The main idea of AI Engineering Buildcamp is that you learn AI engineering by building one system step by step. Over the course of the program, you are working on three types of projects: the module projects, the homework projects, and your own capstone.

1. Module Projects
Inside each module, you’ll learn concepts through hands-on projects, giving you the chance to try out new patterns or tools applied to real use cases.




These module projects become examples you can study, reuse, and adapt for your own work:

Module 1 introduces the foundation through the Documentation Agent project. You learn the basic architecture of an AI application that works with external knowledge: LLMs, the OpenAI API, RAG, chunking, indexing, and retrieval.

Module 2 builds on that foundation by expanding into practical retrieval workflows. This is where projects like the FAQ Assistant, YouTube Transcript Summarizer, and PDF Book Processor show how the same RAG principles apply to different kinds of data.

Module 3 adds agent capabilities. The course projects move beyond retrieval and start showing how to build systems that can use tools, perform web research, and continue based on intermediate results. You build projects like the Tool-Using AI Agent and Web and YouTube Researcher.

Module 4 adds reliability and a more advanced system structure. The focus shifts toward testing agent behavior, evaluating outputs with LLM judges, and working with patterns like routing, pipelines, feedback loops, parallelization, and orchestrator-worker systems.

Module 5 adds observability. You learn how to instrument them with tools like Pydantic Logfire, OpenTelemetry, Jaeger, and lighter-weight dashboard setups.

Module 6 adds systematic evaluation and improvement. You learn how to use offline evaluation, evaluation datasets, judge alignment, and synthetic data for agent evaluation.

Modules 7 and 8 extend the earlier building blocks into more advanced systems such as the Coding Agent, Code Analysis Agent, and Deep Research Agent. You see what happens when retrieval, tool use, and evaluation are combined into larger applications.

Module 9 and beyond bring everything together through demos, peer review, and an optional hackathon.

2. Homework Projects

In addition to module projects, most modules end with a separate homework mini-project, where you build another system yourself and practice the same ideas in a new setting.

Module 1: Document processing with AI. You download books, extract PDF text, chunk documents, and build a full RAG pipeline.

Module 3: Wikipedia Agent. You implement search and page-fetching tools and build an agent using a framework of your choice.

Module 4: Testing and evaluation through the DuckDB SQL Agent. You build an agent that queries NYC taxi data, write pytest tests, add LLM judges, and track costs.

Module 5: Trivia Quizmaster Agent. You build an interactive trivia system and instrument it with Logfire.

Module 6: Systematic evaluation through the Recipe Assistant Evaluation: You design 20+ evaluation scenarios, run batch tests, and detect hallucinations.

3. Capstone Project
On top of that, you’ll be working on your capstone project during the course. Right from the start, you’ll pick a use case to build your project around.




As you go through the course, you’ll keep adding to it, layer by layer, each module bringing something new to your project:

Module 1: You define the use case and build the first working version using LLMs, RAG, chunking, indexing, and retrieval.

Module 2: You make your capstone more useful and realistic by adapting it to your own data sources, search needs, and retrieval design.

Module 3: Your project moves beyond answering with retrieved context and becomes a system that can use tools, make intermediate decisions, and continue based on what it discovers.

Module 4: You add testing, evaluate outputs more systematically, and start thinking about architecture patterns that make the system more robust.

Module 5: You instrument your capstone with monitoring and observability so you can inspect what it is doing and better understand failures.

Module 6: You evaluate the project using scenarios, datasets, and structured quality checks to improve it based on evidence.

Modules 7 and 8: You spend more time strengthening, extending, and deploying the project.

Module 9 and beyond: You present the result, get peer feedback, and optionally continue developing it further in the hackathon setting.

So while the module projects give you examples and the homework projects give you practice, the capstone gives you one continuous system that grows with the course from a basic RAG application into a much more complete AI product.
